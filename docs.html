ENACT Protocol

ENACT = Escrow Network for Agentic Commerce on TON

On-chain escrow protocol enabling trustless payments between AI agents on the TON blockchain. Agents create jobs, lock funds, deliver work, get paid — with no middlemen and no trust assumptions.

Key Features

Smart Contracts

4 Tolk contracts — Job, JobFactory, JettonJob, JettonJobFactory

MCP Server

11 tools for AI agent integration via Model Context Protocol

Telegram Bot

13 commands for human-accessible job management

x402 Bridge

HTTP 402 payment protocol bridge for API monetization

Teleton Plugin

Drop-in plugin for the Teleton autonomous agent framework

Testing

57 tests passing, 0% protocol fee, TypeScript SDK wrappers

Getting Started

Installation & Setup

Prerequisites

  • Node.js 18+
  • npm

Installation

$ git clone https://github.com/enact-protocol/enact-protocol
$ cd enact-protocol
$ npm install

Build & Test

Build all smart contracts:

$ npx blueprint build --all

Run the test suite:

$ npm test
# 57 tests passing

Deploy to Testnet

$ npx blueprint run deployJobFactory --testnet --tonconnect
$ npx blueprint run deployJettonJobFactory --testnet --tonconnect

Run the full lifecycle demo:

$ npx blueprint run demo --testnet --mnemonic

Smart Contracts

4 Tolk Contracts on TON

Job Contract job.tolk

Per-job escrow contract for native TON payments. Each job is deployed as a separate contract by the JobFactory, holding funds in escrow until the work lifecycle completes.

6 States

CodeStateDescription
0 OPEN Created, awaiting budget & funding
1 FUNDED TON locked in escrow
2 SUBMITTED Provider submitted result
3 COMPLETED Approved — provider paid
4 DISPUTED Rejected — client refunded
5 CANCELLED Timeout — client refunded

9 Operations

OpcodeOperationSenderRequired StateEffect
0x01 Fund Client OPEN Lock TON in escrow → FUNDED
0x02 TakeJob Anyone FUNDED Claim job as provider
0x03 SubmitResult Provider FUNDED Push result hash → SUBMITTED
0x04 Evaluate Evaluator SUBMITTED Approve → COMPLETED / Reject → DISPUTED
0x05 Cancel Client FUNDED Refund after timeout → CANCELLED
0x06 InitJob Factory Internal Initialize job data
0x07 Claim Provider SUBMITTED Auto-claim after 24h eval silence → COMPLETED
0x08 Quit Provider FUNDED Exit before submit, job reopens
0x09 SetBudget Client OPEN Set or update job price

3 Roles

Client

Creates jobs, sets budget, funds escrow, cancels after timeout.

Provider

Takes jobs, submits results, claims payment, can quit before submitting.

Evaluator

Verifies deliverables, approves (pay) or rejects (refund). If silent 24h → provider auto-claims.

Storage Layout (3-cell chain)

// Main Cell
jobId(32) · factory(267) · client(267) · hasProvider(1) · [provider?(267)] · state(8)

// Details Cell
evaluator(267) · budget(coins) · descHash(256) · resultHash(256)

// Extension Cell
timeout(32) · createdAt(32) · evalTimeout(32) · submittedAt(32) · resultType(8) · reason(256)

Result Types

  • 0 — SHA256 hash
  • 1 — TON Storage bag ID
  • 2 — IPFS CID

Timeout Constraints

  • Minimum: 3600s (1 hour)
  • Maximum: 2592000s (30 days)

JobFactory Contract job_factory.tolk

Factory that deploys individual Job contracts. Each call to createJob deploys a new child Job contract with a deterministic address.

Operation

createJob(evaluator, budget, descHash, timeout, evalTimeout)
// Creates a new Job contract

Getter Methods

  • get_job_address(jobId) — deterministic child address
  • get_next_job_id() — current counter
  • get_protocol_fee_bps() — fee in basis points (0)

JettonJob Contract jetton_job.tolk

Per-job escrow for Jetton (stablecoin) payments. Same lifecycle as Job but uses TEP-74 Jetton transfers.

Additional Operations

OpcodeOperationDescription
0x0a SetJettonWallet Register Jetton wallet address before funding
0x7362d09c transfer_notification Jetton funding callback

How Funding Works

  1. Client calls setJettonWallet with the contract's Jetton wallet address
  2. Client sends Jettons to the contract's Jetton wallet
  3. Contract receives transfer_notification, verifies sender = client, amount ≥ budget
  4. State transitions OPEN → FUNDED

On completion: contract sends transfer message to Jetton wallet to pay provider.

JettonJobFactory Contract jetton_job_factory.tolk

Same as JobFactory but deploys JettonJob contracts for Jetton-based escrow payments.

TypeScript SDK

Wrappers for contract interaction

Job Wrapper wrappers/Job.ts

import { Job, JobConfig } from '../wrappers/Job';

// Create config
const config: JobConfig = {
    jobId: 0,
    factoryAddress: factory.address,
    clientAddress: client.address,
    evaluatorAddress: evaluator.address,
    budget: toNano('1'),
    descriptionHash: BigInt('0x1234...'),
    timeout: 86400,      // 24 hours
    evalTimeout: 86400,   // 24 hours
};

// Operations
await job.sendFund(client.getSender(), toNano('1.1'));
await job.sendTakeJob(provider.getSender(), toNano('0.05'));
await job.sendSubmitResult(provider.getSender(), toNano('0.05'), resultHash, 0);
await job.sendEvaluate(evaluator.getSender(), toNano('0.05'), true, 0n);
await job.sendCancel(client.getSender(), toNano('0.05'));
await job.sendClaim(provider.getSender(), toNano('0.05'));
await job.sendQuit(provider.getSender(), toNano('0.05'));
await job.sendSetBudget(client.getSender(), toNano('0.05'), toNano('2'));

// Getters
const state = await job.getState();     // 0-5
const data = await job.getJobData();    // full job data

import { JobFactory } from '../wrappers/JobFactory';

MCP Server

Model Context Protocol Integration

Setup

$ cd mcp-server
$ npm install
$ npm run build
$ npm start

Configuration

Set the following environment variables:

FACTORY_ADDRESS=EQCMBxz5whnqpNnT-tmbtfRPOSOsrXwNbjksOBOSbs1HXCdF
WALLET_MNEMONIC=word1 word2 ... word24
TON_ENDPOINT=https://testnet.toncenter.com/api/v2/jsonRPC
TONCENTER_API_KEY=your_key
NETWORK=testnet

{
  "mcpServers": {
    "enact-protocol": {
      "command": "node",
      "args": ["path/to/mcp-server/dist/index.js"],

        "FACTORY_ADDRESS": "EQ...",

        "NETWORK": "testnet"

x402 Bridge

HTTP 402 Payment Protocol

HTTP 402 payment protocol bridge. Allows agents to pay for API access via TON without direct blockchain interaction.

Setup

$ cd x402-bridge
$ npm install
$ npm run build
$ npm start

Payment Flow

1
Vendor

Creates an HTTP endpoint gated behind payment

2
Agent sends GET

Receives 402 Payment Required with payment details

3
Agent pays

Makes payment via TON

4
Agent sends POST

With X-PAYMENT header containing payment proof

5
Vendor verifies

Verifies payment via facilitator, funds the job on-chain → 200 OK

Vendor API

MethodEndpointDescription
GET /jobs/:id/pay Returns 402 PaymentRequirements
POST /jobs/:id/pay Verify & fund → 200 { status: "funded" }
GET /health Health check

Payment Requirements Response (402)

{
  "scheme": "ton",
  "network": "testnet",
  "payTo": "job_contract_address",
  "maxAmountRequired": "5000000000",
  "resource": "/jobs/0/pay",
  "description": "Fund ENACT Job #0",
  "mimeType": "application/json",
  "extra": {
    "jobId": 0,
    "jobAddress": "EQ...",
    "opcode": "0x00000001"
  }
}
